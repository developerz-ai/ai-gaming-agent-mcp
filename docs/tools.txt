AI GAMING AGENT - MCP Tools Reference
=======================================

SCREEN TOOLS
------------

screenshot
  Description: Capture current screen
  Parameters:
    monitor (optional) - Monitor index for multi-monitor setups
  Returns: Base64 encoded PNG image

get_screen_size
  Description: Get screen dimensions
  Parameters: None
  Returns: Width and height in pixels

analyze_screen
  Description: Use local VLM to analyze screen content
  Parameters:
    prompt - Question about the screen content
  Returns: VLM response text


MOUSE TOOLS
-----------

click
  Description: Click at screen coordinates
  Parameters:
    x - X coordinate (pixels from left)
    y - Y coordinate (pixels from top)
    button (optional) - left, right, or middle
  Returns: Success or failure status

double_click
  Description: Double-click at coordinates
  Parameters:
    x - X coordinate
    y - Y coordinate
  Returns: Success or failure status

move_to
  Description: Move mouse cursor
  Parameters:
    x - Target X coordinate
    y - Target Y coordinate
    duration (optional) - Movement time in seconds
  Returns: Success or failure status

drag_to
  Description: Drag from current position to target
  Parameters:
    x - Target X coordinate
    y - Target Y coordinate
    duration (optional) - Drag duration in seconds
  Returns: Success or failure status

scroll
  Description: Scroll mouse wheel
  Parameters:
    clicks - Number of scroll ticks (positive=up, negative=down)
    x (optional) - Scroll at specific X position
    y (optional) - Scroll at specific Y position
  Returns: Success or failure status

get_mouse_position
  Description: Get current mouse cursor location
  Parameters: None
  Returns: X and Y coordinates


KEYBOARD TOOLS
--------------

type_text
  Description: Type a string of text
  Parameters:
    text - Text to type
    interval (optional) - Delay between keystrokes
  Returns: Success or failure status

press_key
  Description: Press a single keyboard key
  Parameters:
    key - Key name (e.g., enter, escape, tab, f1)
    modifiers (optional) - List of modifier keys (ctrl, alt, shift)
  Returns: Success or failure status

hotkey
  Description: Press a key combination
  Parameters:
    keys - List of keys to press together (e.g., ctrl, c)
  Returns: Success or failure status


FILE TOOLS
----------

read_file
  Description: Read file contents
  Parameters:
    path - File path (must be in allowed directories)
  Returns: File contents (text or base64 for binary)

write_file
  Description: Write content to file
  Parameters:
    path - File path
    content - Content to write
    binary (optional) - Treat content as base64 binary
  Returns: Success or failure status

list_files
  Description: List directory contents
  Parameters:
    path - Directory path
  Returns: List of file and folder names

upload_file
  Description: Upload file to gaming PC
  Parameters:
    path - Destination path
    content - File content (text or base64)
    binary (optional) - Content is base64 binary
  Returns: Success or failure status

download_file
  Description: Download file from gaming PC
  Parameters:
    path - File path to download
  Returns: Base64 encoded file content


SYSTEM TOOLS
------------

execute_command
  Description: Run shell/terminal command
  Parameters:
    command - Command to execute
    timeout (optional) - Max execution time in seconds
  Returns: stdout, stderr, and exit code

get_system_info
  Description: Get system resource usage
  Parameters: None
  Returns: CPU, RAM, and GPU usage statistics

list_windows
  Description: List all open windows
  Parameters: None
  Returns: Window titles and handles

focus_window
  Description: Bring window to foreground
  Parameters:
    title - Window title (partial match)
    OR handle - Window handle ID
  Returns: Success or failure status


WORKFLOW TOOLS
--------------

run_workflow
  Description: Execute a sequence of tool actions as a single workflow
  Parameters:
    steps - List of step dictionaries, each containing:
      - tool (str, required) - Name of the tool to call
      - args (dict, optional) - Arguments to pass to the tool
      - wait_ms (int, optional) - Milliseconds to wait after this step
      - description (str, optional) - Human-readable step description
      - continue_on_error (bool, optional) - Continue if this step fails
  Returns: Dictionary with:
    - success - True if all steps succeeded
    - total_steps - Number of steps in workflow
    - completed_steps - Number of steps that completed
    - failed_step - Index of first failed step (if any)
    - results - List of results from each step
    - total_time_ms - Total execution time
    - error - Error message if workflow failed
  Example:
    steps = [
      {"tool": "execute_command", "args": {"command": "gnome-terminal"}, "wait_ms": 1000},
      {"tool": "type_text", "args": {"text": "echo hello"}, "wait_ms": 100},
      {"tool": "press_key", "args": {"key": "enter"}}
    ]

demo_terminal_workflow
  Description: Complete terminal workflow - open, type, execute, capture, close
  Parameters:
    text (optional) - Text/command to type (default: "echo hello world")
    terminal_wait_ms (optional) - Wait time for terminal to open (default: 2000)
    post_type_wait_ms (optional) - Wait time after typing (default: 500)
    post_enter_wait_ms (optional) - Wait time after pressing Enter (default: 1000)
    capture_screenshot (optional) - Take screenshot after execution (default: true)
    close_terminal (optional) - Close terminal at the end (default: true)
  Returns: Dictionary with:
    - success - True if all steps completed
    - terminal_command - Terminal application used
    - platform - Operating system
    - text_typed - The text that was typed
    - screenshot - Screenshot result (if captured)
    - steps_completed - List of completed step names
    - total_time_ms - Total execution time
    - error - Error message if any step failed
  Notes:
    - Auto-detects appropriate terminal for platform (Linux/macOS/Windows)
    - Complete automation example in a single call
    - Useful for testing and demonstrations


EXPOSED RESOURCES
-----------------
Resources can be accessed by URI:

  screen://current   - Live screenshot (refreshes each access)
  file:///{path}     - File system access
  system://info      - System information
  game://status      - Custom game state (if implemented)
